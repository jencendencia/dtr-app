# GitHub Release Upload Guide

This guide documents the correct process for uploading the DTR app installer to GitHub releases, avoiding the filename mismatch error.

## The Problem

When electron-builder generates the installer, it creates files with **spaces** in the name:
```
Biometric DTR System Setup 1.2.4.exe
```

But when uploaded to GitHub, spaces are converted to **dots**:
```
Biometric.DTR.System.Setup.1.2.4.exe
```

The `latest.yml` file generated by electron-builder references the original filename (with spaces/hyphens), causing a **404 error** when the auto-updater tries to download the update.

## Step-by-Step Process

### 1. Bump Version

Edit `package.json` and update the version:
```json
{
  "version": "1.2.5"
}
```

### 2. Build the Installer

```bash
npm run dist
```

This creates files in the `dist/` folder:
- `Biometric DTR System Setup X.X.X.exe`
- `Biometric DTR System Setup X.X.X.exe.blockmap`
- `latest.yml`

### 3. Fix the latest.yml (CRITICAL)

Open `dist/latest.yml` and replace **hyphens with dots** in the filename:

**Before (wrong):**
```yaml
version: 1.2.5
files:
  - url: Biometric-DTR-System-Setup-1.2.5.exe
    sha512: ...
    size: ...
path: Biometric-DTR-System-Setup-1.2.5.exe
sha512: ...
releaseDate: '...'
```

**After (correct):**
```yaml
version: 1.2.5
files:
  - url: Biometric.DTR.System.Setup.1.2.5.exe
    sha512: ...
    size: ...
path: Biometric.DTR.System.Setup.1.2.5.exe
sha512: ...
releaseDate: '...'
```

### 4. Delete Old Release (if exists)

```bash
gh release delete vX.X.X --repo jencendencia/dtr-app --yes
```

### 5. Create New Release and Upload

```bash
gh release create vX.X.X ^
  "dist\Biometric DTR System Setup X.X.X.exe" ^
  "dist\Biometric DTR System Setup X.X.X.exe.blockmap" ^
  "dist\latest.yml" ^
  --repo jencendencia/dtr-app ^
  --title "vX.X.X" ^
  --notes "Release notes here"
```

### 6. Verify the Upload

```bash
gh release view vX.X.X --repo jencendencia/dtr-app
```

Check that:
- Asset names use **dots** (not hyphens or spaces)
- `latest.yml` references the correct filename with dots

### 7. Verify latest.yml Content

```bash
# Download and check the latest.yml from the release
# It should reference Biometric.DTR.System.Setup.X.X.X.exe
```

## Quick Reference Checklist

- [ ] Version bumped in `package.json`
- [ ] Ran `npm run dist`
- [ ] Fixed `latest.yml` - replaced hyphens with dots in filename
- [ ] Deleted old release (if exists)
- [ ] Created new release with all 3 files:
  - `.exe` installer
  - `.exe.blockmap` file
  - `latest.yml` (with dots in filename)
- [ ] Verified asset names use dots
- [ ] Verified `latest.yml` content matches asset names

## Token Management

**For public repos:** No token needed for auto-updater.

**For private repos:** Pass token via environment variable when building:
```bash
$env:GH_TOKEN = "ghp_your_token_here"
npm run dist
```

**Never commit tokens to source code** - GitHub's secret scanning will block the push.

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| 404 when downloading update | `latest.yml` filename doesn't match asset | Fix `latest.yml` to use dots |
| Push rejected (secret scanning) | Token in committed code | Remove token, use env variable |
| Release not found | Old release not deleted | Delete old release first |
