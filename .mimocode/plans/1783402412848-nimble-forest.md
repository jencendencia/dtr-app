# Plan: Add Bulk Delete via Checkboxes to Attendance Records Table

## Context
In the Search Teacher tab, the attendance records table currently has a per-row Delete button. To delete multiple days, the user must click Delete on each row individually. This plan adds checkboxes so the user can select multiple days and delete them in one action.

## File to Modify
- `E:\Joel\DTR-app\src\renderer\renderer.js` — the `displayTeacherLogs()` function (lines 1241-1461)

No backend changes needed — the existing `delete-attendance-log` IPC handler already supports per-log deletion, and we'll loop through selected days just like the current per-day delete does.

## Changes (all within `displayTeacherLogs()`)

### 1. Add a "Delete Selected" button above the table (~line 1277)
- Insert a container div before the `<table>` with:
  - A "Delete Selected" button (red, same style as existing delete buttons)
  - A counter showing number of selected days (e.g. "3 days selected")
- The button and counter are hidden by default, shown when ≥1 checkbox is checked

### 2. Add checkbox column to table header (line 1278)
- Add a new `<th>` at the start of the header row with a "Select All" checkbox
- Style: centered, same padding as other headers

### 3. Add checkbox cell to each table row (lines 1402-1413)
- Add a `<td>` at the start of each row with a checkbox input
- Each checkbox has `data-day="${i}"` to identify which day it corresponds to
- Checkboxes for holiday/full-suspension rows are disabled (can't delete holidays)
- Rows with no logs (skipped via `continue` on line 1294) naturally won't have checkboxes

### 4. Add event handlers (after line 1460, before the closing `}`)

**Select All handler:**
- Toggle all non-disabled row checkboxes when the header checkbox is clicked
- Update the counter and button visibility

**Individual checkbox handler:**
- On each checkbox change, update the counter and show/hide the Delete Selected button
- Keep the Select All checkbox in sync (checked if all are checked, indeterminate if some)

**Delete Selected button handler:**
- Collect all checked day numbers
- Show confirmation: "Are you sure you want to delete all logs for X selected days?"
- Skip any holiday days (same protection as existing delete)
- Loop through each selected day's logs and call `ipcRenderer.invoke('delete-attendance-log', log.id)` for each
- Show toast: "Logs deleted successfully"
- Re-fetch fresh data and re-render the table (same pattern as existing per-day delete on lines 1454-1457)

6. Select 2-3 days, click "Delete Selected", confirm — all logs for those days should be deleted
7. Verify the table refreshes and deleted days no longer show logs
8. Verify individual per-row Delete button still works as before
