# Plan: Add Device Role to Teacher Enrollment

## Summary

Add a "Device Role" (Admin / User) field to the teacher enrollment flow. When enrolling a teacher to the ZKTeco device, the role determines their privilege level on the device (0 = normal user, 1 = admin). This is stored in the Teachers table so it persists across enrollments.

## Changes

### 1. Add `device_role` column to Teachers table

**File:** `src/db/connection.js` — In the schema creation, add the column to the `CREATE TABLE` statement:

```sql
device_role INTEGER DEFAULT 0
```

Also add an `ALTER TABLE` migration for existing databases:
```sql
ALTER TABLE Teachers ADD COLUMN device_role INTEGER DEFAULT 0
```

### 2. Update `add-teacher` handler

**File:** `src/main/main.js` — Accept `device_role` in the input and store it:
```js
const { name, biometric_id, device_role } = teacher;
// INSERT INTO Teachers (name, biometric_id, device_role) VALUES (?, ?, ?)
```

### 3. Update `update-teacher` handler

**File:** `src/main/main.js` — Accept `device_role` and update it:
```js
db.prepare('UPDATE Teachers SET name = ?, biometric_id = ?, device_role = ? WHERE id = ?')
```

### 4. Update enrollment handlers to use stored role

**File:** `src/main/main.js` — In `enroll-teacher-to-device`:
```js
const teacher = db.prepare('SELECT id, name, biometric_id, device_role FROM Teachers WHERE id = ?').get(teacherId);
const result = await zktecoService.setUser(teacher.id, teacher.biometric_id, teacher.name, '', teacher.device_role || 0, 0);
```

In `enroll-all-teachers-to-device`:
```js
const teachers = db.prepare('SELECT id, name, biometric_id, device_role FROM Teachers WHERE status = ?').all('active');
// Use t.device_role || 0 when calling setUser
```
