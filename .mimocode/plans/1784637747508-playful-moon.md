# Fix ZKTeco "rawLogs is not iterable" Sync Error

## Root Cause Analysis

The `zkteco-js` library returns data in a different shape than our `zktecoService.js` expects. There are **4 bugs** in `src/main/zktecoService.js`:

### Bug 1: `getAttendances()` returns `{ data: [...] }`, not an array
- **Library** (`ztcp.js:544`): `return { data: records };`
- **Our code** (`zktecoService.js:97`): `const rawLogs = await this.device.getAttendances();`
- **Our code** (`zktecoService.js:98`): `rawLogs.length` → `undefined` (object has no `.length`)
- **Our code** (`zktecoService.js:102`): `for (const log of rawLogs)` → **"rawLogs is not iterable"**

### Bug 2: `getUsers()` returns `{ data: [...] }`, not an array
- **Library** (`ztcp.js:490`): `return { data: users };`
- **Our code** (`zktecoService.js:165`): `const users = await this.device.getUsers();`
- **Our code** (`zktecoService.js:166`): `users.length` → `undefined`

### Bug 3: `getInfo()` returns different field names
- **Library** (`ztcp.js:608-611`): `{ userCounts, logCounts, logCapacity }`
- **Our code** (`zktecoService.js:35-37`): reads `info.userCount`, `info.attendanceCount`, `info.attendanceCapacity`

### Bug 4: Attendance record field names don't match
- **Library** (`utils.js:153-164`): `{ sn, user_id, record_time, type, state }`
- **Our code** (`zktecoService.js:103-104`): expects `{ id, userId, timestamp, verifyMode, inOutMode, workCode }`
- Also: `record_time` is a string (Date.toString()), not a Date object

## Fix: `src/main/zktecoService.js`

### 1. Fix `connect()` — `getInfo()` field names (lines 31-38)
```javascript
// BEFORE
userCount: info.userCount || 0,
attendanceCount: info.attendanceCount || 0,
attendanceCapacity: info.attendanceCapacity || 0

// AFTER
userCount: info.userCounts || 0,
attendanceCount: info.logCounts || 0,
attendanceCapacity: info.logCapacity || 0
```

### 2. Fix `getAttendanceLogs()` — unwrap `{ data: [...] }` and fix field names (lines 97-144)
```javascript
// BEFORE
const rawLogs = await this.device.getAttendances();
console.log(`[ZktecoService] Retrieved ${rawLogs.length} raw attendance log(s)`);
for (const log of rawLogs) {
  // expects: log.userId, log.timestamp, log.inOutMode, etc.
}

// AFTER
const response = await this.device.getAttendances();
const rawLogs = response.data || [];
console.log(`[ZktecoService] Retrieved ${rawLogs.length} raw attendance log(s)`);
for (const log of rawLogs) {
  // actual fields: log.user_id, log.record_time, log.type, log.state
}
```

The record field mapping becomes:
- `log.user_id` → `employeeId`
- `log.record_time` → `logTime` (already a string from Date.toString(), need to parse to "YYYY-MM-DD HH:MM:SS")
- `log.type` → used for check-in/check-out (0 = check-in, 1 = check-out based on ZKTeco protocol)
- `log.state` → verification mode

### 3. Fix `getUsers()` — unwrap `{ data: [...] }` (lines 165-167)
```javascript
// BEFORE
const users = await this.device.getUsers();
console.log(`[ZktecoService] Retrieved ${users.length} user(s)...`);
return { success: true, data: users, ... };

// AFTER
const response = await this.device.getUsers();
const users = response.data || [];
console.log(`[ZktecoService] Retrieved ${users.length} user(s)...`);
return { success: true, data: users, ... };
```
